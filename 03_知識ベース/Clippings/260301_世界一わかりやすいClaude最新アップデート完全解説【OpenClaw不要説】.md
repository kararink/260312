# ClaudeがAIエージェントに進化した

今回のアップデートを一言でまとめると、Claudeが「チャットAI」から**AIエージェント**に本格的に進化したということです。

具体的には3つの大きなアップデートがあります。

- **Auto Memory** → AIが自分で学習して記憶する
- **Remote Control** → スマホからPCのClaude Codeを操作できる
- **Cowork Scheduled Tasks** → 指定した時間に自動でタスクを実行する

これ、3つ合わせると何が起こるか。

OpenClawっぽい汎用性の高いAIエージェントを、Claude公式の機能とサブスクだけで構築できる可能性が出てきたんです。

マジでこれはデカい。

# そもそも今Claudeに何が起きているのか

まず全体像を整理します。

Anthropicは2026年に入ってから、Claude Codeを**開発ツール**から**汎用AIエージェント基盤**に進化させようとしています。

Claude Code自体はCLIベースのコーディングエージェントなんですが、最近のアップデートを見ると明らかにコーディング以外の用途も想定している。

コンテンツ制作、リサーチ、ファイル操作、API連携。

こういったことを全部Claude Codeでやれるようにしていこうという設計思想が見えてきます。

そしてその核になるのが、今回紹介する3つのアップデートです。

# アップデート① AIが自分で学んで覚えるAuto Memory機能

まず最初のアップデート、 **Auto Memory** から解説します。

これは要するに、Claude Codeが**自分でメモを取って記憶する**機能です。

今までもClaude Codeには [CLAUDE.md](https://claude.md/) というファイルがあって、そこにユーザーが手動でルールや好みを書いておく仕組みはありました。

でもAuto Memoryはこれとは別物。

AIが自分で [MEMORY.md](https://memory.md/) というファイルを作って、プロジェクトのパターンやデバッグのコツ、あなたの好みなどを勝手に記録していくんです。

## Auto Memoryが記憶するもの

具体的にどんなことを覚えてくれるかというと、

- ワークフロー
- ツールの選び方
- 重要ファイル
- コミュニケーションスタイル
- 厄介な問題の解決策

など、重要な情報を覚えておいてくれます。でかいんですよこれが。

これ、使えば使うほどClaude Codeが賢くなっていくということです。

しかも [MEMORY.md](https://memory.md/) の最初の200行は、毎回セッション開始時に自動で読み込まれる。

つまり、昨日話したことを今日も覚えているし、先週ハマったバグの解決策も記憶している。

毎回ゼロから説明する必要がなくなるんです。

## CLAUDE.mdとの違い

ここ、ちょっとややこしいので整理します。

- [CLAUDE.md](https://claude.md/) → ユーザーが手動で書くルール・指示書。チームで共有する用途にも使える。
- [MEMORY.md](https://memory.md/) → AIが自動で書く学習メモ。個人のマシンにローカル保存される。

両方使えるのがポイントです。

[CLAUDE.md](https://claude.md/) でプロジェクトのルールを明示しつつ、 [MEMORY.md](https://memory.md/) でAIが日々学習した知見を積み上げていく。

この組み合わせが強い。

ちなみに /memory コマンドで記憶の中身を確認・編集できるし、設定でオフにすることもできます。

# アップデート② スマホからAIエージェントを操る Remote Control機能

次に**Remote Control**。

これは、PCで動いているClaude Codeを**スマホやタブレットから操作できる**機能です。

claude remote-control とターミナルで打つだけで、QRコードが表示される。 それをスマホで読み取ると、[Claude.ai](https://claude.ai/)やClaudeアプリからそのセッションに接続できる。

外出先からスマホで指示を出すだけで、家のパソコンでClaude Codeが動いてくれるということです。

## Remote Controlの何がすごいのか

「ブラウザからClaude使えるじゃん」って思いますよね。

でもRemote Controlは全然違います。

ブラウザ版のClaudeは、Claudeのサーバー上で動くクラウド環境。 Remote Controlは、あなたのPCのローカル環境そのものを操作する。

つまり。

- あなたのPCのファイルにそのまままアクセスできる
- プロジェクトの設定もそのまま反映される
- ターミナルとスマホの会話がリアルタイムで同期する

これって、構造としては**OpenClawと全く同じ**なんですよ。

OpenClawはDiscordやTelegramを通してPCのAIエージェントを操作する。 Remote Controlは、Claudeアプリを通してPCのClaude Codeを操作する。

やっていることは本質的に同じ。

## Remote Controlの始め方

始め方は超シンプル。

**ステップ①** ターミナルで claude remote-control を実行

**ステップ②** 表示されたURLにアクセスするか、QRコードをスキャン

**ステップ③** [claude.ai/code](https://claude.ai/code) またはClaudeアプリでセッションに接続

これだけ。

ただし現時点ではMaxプラン（月額$100〜）が必要です。 Proプラン対応は近日中とのこと。

また、PCのターミナルは開きっぱなしにしておく必要があります。 PCがスリープしたりネットが10分以上切れるとセッションが終了する点には注意が必要です。

# アップデート③ AIが定期作業を自動実行 Cowork Scheduled Tasks機能

そして3つ目、 **Coworkのスケジュールタスク**機能。

Coworkは、Claude Desktopアプリに搭載されたエージェント機能です。

今回のアップデートで、Coworkに**スケジュールタスク**が追加されました。

これが何かというと、Claudeに「毎朝9時にこれをやって」「毎週金曜にこれをまとめて」と設定しておくと、 **指定した時間に自動でタスクを実行してくれる** 機能です。

公式の例だとこんな使い方が紹介されています。

- 毎朝のブリーフィング作成
- 週次のスプレッドシート更新
- 毎週金曜のチームプレゼン資料作成

これ、要するにClaude版のcronジョブです。

しかもファイルの読み書き、マルチステップの複雑なタスクにも対応している。

## 3つのアップデートを組み合わせるとOpenClawになる

ここからが本題です。

この3つのアップデートを組み合わせると何が起こるか。

- **Auto Memory** → AIが学習し、あなたのやり方を覚える
- **Remote Control** → スマホからどこでも操作できる
- **Scheduled Tasks** → 定期タスクを自動実行する

これ、OpenClawがやっていることとほぼ同じ構造なんですよ。

OpenClawは「スマホから指示 → PCでAIが自動実行 → 学習して精度が上がる」という仕組み。

今回のClaude公式アップデートでも、同じような体験が公式の安定したインフラの上で実現できる可能性が出てきたということです。

# OpenClawとの違いは？勝っているところ・負けているところ

とはいえ、完全にOpenClawの代替になるかというと、まだ微妙なところもあります。

## Claudeが勝っているところ

- 公式サポートがある → 安定性が高い
- セットアップが簡単 → コマンド一発で始められる
- Scheduled Tasksがある → 定期実行が公式機能として使える
- サブスクでOpus 4.6が使える → 日本語の精度が高い

## OpenClawが勝っているところ

- モデルを自由に差し替えられる → GPT、Gemini、Claudeなどを使い分けられる
- プラットフォームの自由度 → Discord、Telegram、LINEなど好きな場所から操作
- コストの柔軟性 → API従量課金なので使った分だけ

ぼく個人の感想としては、OpenClawの自由度は捨てがたい。

Opus 4.6は確かに今は強いです。 でも今後、GPT 5.3とか新しいモデルが出てきたときに、すぐ差し替えられるOpenClawの柔軟性は重要。

ただ、すでにClaude Maxに課金していて、手軽にエージェント的な使い方をしたい人には、今回のアップデートは十分アリだと思います。

# Opus 4.6が使えるのは結構デカい

ここはちゃんと触れておきたいんですが、Opus 4.6が使えるっていうのは結構大きなメリットです。

ぼく自身、ブログ記事とかXポストをAIに書いてもらうことが多いんですが、やっぱりOpus 4.6で生成した方が日本語の精度が明らかに高い。

文章の自然さ、ニュアンスの拾い方、語彙の選び方。

この辺りは今のところOpus 4.6が頭一つ抜けています。

コンテンツ制作をメインにAIエージェントを使いたい人にとっては、Claude Codeという選択肢はかなり有力です。

# つまずきポイント（よくあるハマりどころ）

先に共有しておきます。

**Remote Controlでセッションが切れる** → PCがスリープすると切れます。長時間使うならスリープ無効にしておくなど対応が必要です。

**Scheduled TasksがPCスリープ中に動かない** → PCが起動していて、Claude Desktopアプリが開いているのが条件。スリープ中はスキップされます

**Auto Memoryの内容が意図しないものになる** → /memory コマンドで中身を確認・編集できます。定期的にチェックするのがおすすめ

**Maxプランのコストが高い** → まずはProプランでCoworkのScheduled Tasksから始めてみるのがおすすめ
