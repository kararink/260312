# 📖 カンペ：エージェントスキルの設計規則（Best Practices for Skills）

このファイルは、AIエージェントが「新しいAIスキル（エージェント拡張機能）」の設計図を作成する際に参照する、極秘のリファレンス（カンペ）です。
**Webサイト・アプリ制作の設計図を書く際には、このファイルの内容は無視してください。**

## 1. スキル設計の大原則（単一責任の原則）
- 1つのスキルがカバーするのは「1つのドメイン（役割）」のみ。何でもできる万能スキルを作らないこと。
- 明確な入力（何を受け取るか）と、明確な出力（何を返すか）を定義すること。

## 2. フォルダ構造のベストプラクティス
スキルを設計（ファイル構成を定義）する際は、必ず以下の公式フォルダ構造に準拠するよう設計図に記載すること。

```text
📁 .agent/skills/<skill-name>
 ├── 📄 SKILL.md       （スキルの本体/説明書。エージェントへの指示と実行ワークフローを記載）
 ├── 📁 resources/     （プロンプトファイル、テンプレート、カンペなどのリファレンスを格納）
 ├── 📁 scripts/       （PythonやNode.jsなどの処理スクリプトが必要な場合のみ格納）
 └── 📁 examples/      （使用例やテストデータなどの参照用）
```
※ `prompts/` という名前のフォルダは公式ガイドにないため、プロンプトは `resources/` 内に配置すること。

## 3. 起動トリガーとメタデータ定義（最重要）
設計図の冒頭で、ユーザーの言葉から「AIがこのスキルを選ぶためのスイッチ」であるYAMLフロントマターを必ず設計させること。
- **Skill Name (`name`)**: 英小文字・ケバブケース (例: `data-analyzer`)。
- **Description (`description`)**: 
  - 最初は必ず **3人称・現在形の英語** で記述すること。(例: "Analyzes CSV files and generates report.") 
  - エージェントシステムはこの説明文を見てスキルを呼び出すため、「ユーザーがどんな悩みを言った時にこのスキルに頼るべきか」を想定し、具体的かつ魅力的なキーワードを豊富に入れるよう設計すること。

## 4. SKILL.md の必須要件
設計図には、以下の内容を `SKILL.md` の要件として落とし込むこと。
- **Workflow**: エージェントが上から順に実行すべきステップ（Step 1, Step 2...）を具体的に箇条書きで定義すること。
- **💥 見出し構成の厳格化**: 生み出される `SKILL.md` は、システムの一貫性を保つため、必ず以下の見出し（Heading）階層構造を守るよう設計させること。
  ```markdown
  # [スキル名] (H1)
  ## Workflow (H2)
  ### Step 1: [処理内容] (H3)
  ### Step 2: [処理内容] (H3)
  ## Usage Instructions (H2)
  ```

## 5. スクリプトの安全実行（Safe Execution）
Pythonなどのスクリプトをスキルに組み込む設計の場合、エージェントがソースコードを読まずに済むよう、必ず `argparse` などで `--help` コマンドを実装させ、スキルの最初のステップで `--help` を実行して仕様を自己把握させる設計にすること。
