# フォルダ構造ルール（Folder Structure Rules）
> このルールは、AIがファイルを作成・保存する際に必ず参照すること。

## トップレベルフォルダ一覧

| # | フォルダ名 | 用途 | 書き込み |
|---|---|---|---|
| 1 | `00_システム/` | ユーザー情報の定義やプロンプト・設定・開発ツールなど | ⚠️ 慎重 |
| 2 | `01_プロジェクト/` | 進行中プロジェクトの作業ファイル | ✅ 可 |
| 3 | `02_音声/` | 音声データや文字起こしなどの対象ファイル | ✅ 可 |
| 4 | `03_知識ベース/` | メモやクリッピング、マニュアル、レシピ、人の情報などのインプット | ✅ 可 |
| 5 | `04_アウトプット/` | メルマガ、Xポスト、Note、Instagram などのコンテンツ出力物 | ✅ 可 |
| 6 | `05_日誌/` | デイリージャーナルやWeekly Reviewなどの記録 | ✅ 可 |
| 7 | `99_Sbox/` | 一時メモやドラフト、サンドボックス用途 | ✅ 可 |

## 絶対ルール

1. **ルートディレクトリへのファイル作成 絶対禁止 🚫🚫🚫** — ワークスペース直下にいかなるファイル・フォルダも新規作成してはならない。必ずフォルダ一覧のいずれかの配下に作成すること。一時ファイルが必要な場合は `99_Sbox/` を使用する。判断に迷う場合はユーザーに確認。
2. **新規トップレベルフォルダの作成禁止 🚫** — 一覧にないフォルダを勝手に作ってはならない。新しいカテゴリが必要な場合は既存フォルダのサブフォルダとして作成。迷ったらユーザーに確認。
3. **触ってはいけないフォルダ 🚫** — `.agent`, `.agents`, `.git`, `.obsidian`, `.venv`, `.windsurf` 等のシステム管理・隠しフォルダはAIが直接操作してはならない（ただし `.agent/rules/` や `.agent/workflows/` 等のルール・ワークフロー定義ファイルの更新は、ユーザーの指示がある場合のみ許可）。
4. **ファイルの保存先ルール（ファイル種別 → 保存先マッピング）**
   - **コンテンツ出力**: メルマガ → `04_アウトプット/00_メルマガ/`、Xポスト → `04_アウトプット/01_Xポスト/`、Note → `04_アウトプット/03_Note/`、Note画像 → `04_アウトプット/04_Note画像/`、Instagram → `04_アウトプット/05_Instagram/`
   - **インプット・ナレッジ**: メモ・学習ノート → `03_知識ベース/`、Webスクラップ → `03_知識ベース/Clippings/`、人物情報 → `03_知識ベース/人の情報（relationship）/`、マニュアル → `03_知識ベース/マニュアル/`、文字起こしログ → `03_知識ベース/00_文字起こしログ/`
   - **プロジェクト作業ファイル**: 進行中プロジェクトに紐づくファイルは `01_プロジェクト/` 配下の該当プロジェクトフォルダ
   - **その他**: 日誌・ジャーナル → `05_日誌/`、一時メモやドラフト → `99_Sbox/`、音声データ → `02_音声/`。不明なファイル種別は「ユーザーに確認する」。
5. **ファイル命名規則** 
   - 全てのファイルは原則として **日付付き (`YYYYMMDD_内容.md`)** で統一する。（※例: 20260308_ルールのドラフト.md）
   - ファイル名にスペースは使わず、区切りは `_`（アンダースコア）を使用。
6. **フォルダ構造のガイドライン**
   - 階層の深さは3〜4層まで（深すぎるとトークンを無駄に消費する）
   - 1フォルダ内のファイル数は20件以下を目安（超えたらサブフォルダで分割を検討）
7. **フォルダ名の重複チェック** — ファイルを新規作成する際は既存のフォルダ構造を確認してから配置先を決定。類似名のフォルダがあればそちらを使用。同じ用途のサブフォルダを複数作成しないこと。

## 書き込み注意フォルダ
- `00_システム/` — ユーザーの基本設定、プロンプト、システムツールなど、ワークスペースの根幹に関わるファイルが含まれており、AIの動作やコンテキストに直結するため。

## ルート直下の既知ファイル（レガシー）
> 以下のファイルは初期セットアップ時に作成されたものであり、現在もルートに存在する。
> これらは例外的に許容するが、**新たにルート直下にファイルを作成してはならない**。

- `00_はじめに実行する.md` — 初期セットアップ手順
- `01_名前更新.md` — 名前設定手順
- `02_コンテキスト注入.md` — コンテキスト設定手順
- `03_スキルと文体の学習.md` — スキル学習手順
- `.env` / `.gitignore` — 環境設定ファイル（システム用）

## 判断フローチャート
ファイルの保存先を判断する際は、上から順にチェックし、最初に該当したカテゴリの保存先に配置する:
1. コンテンツの完成物か？ → `04_アウトプット/` 以下の該当サブフォルダ
2. インプット・学習素材か？ → `03_知識ベース/` 以下の該当サブフォルダ
3. 特定プロジェクトの作業ファイルか？ → `01_プロジェクト/` 以下の該当プロジェクトフォルダ
4. 日誌・ジャーナルか？ → `05_日誌/` 以下
5. 音声の文字起こしや関連データか？ → `02_音声/` 以下
6. 一時的なメモや下書きか？ → `99_Sbox/` 以下
最後. 上記どれにも当てはまらない → ユーザーに確認する（勝手に保存しない）
