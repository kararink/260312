## 執筆ルール

### 文体

1. **リツトの話し方に寄せる**: `ritsuto_writer` で取得した文体コンテキストを適用する
2. **口語体で書く**: 書き言葉ではなく、実際に口に出して読める自然な話し言葉で書く
3. **一文を短くする**: 一文は40文字以内を目安にする。息継ぎポイントを意識する
4. **聴衆接続フレーズを多用する**:
   - 「〜じゃないですか」
   - 「覚えてますか？」
   - 「ここ、ちょっと興味ないですか？」
   - 「これ、めちゃくちゃ重要なんですけど」
   - 「どっちがいいですか？」
   - 「ありがとうございます」（挙手や反応の後）
5. **間（ま）を意識する**: 重要な発言の前後に「…」で間を入れる。聴衆が考える時間を作る
6. **感情の起伏を作る**: 共感パート（静）→ 気づきパート（動）→ 衝撃の事実（静→動）のリズムを作る

### レベル調整

1. **専門用語は絶対に使わない**: もし必要な場合は「○○っていうのは△△のことです」と即座に噛み砕く
2. **比喩を多用する**: 難しい概念は日常の比喩（お水のパッケージ、レストランのメニュー等）で説明する

> **❗ 比喩の論理一貫性ルール**: 比喩を書くときは以下の3条件を必ず満たすこと。
> 1. **一本線テスト**: 比喩の「導入→展開→着地」が1つのロジックで一直線に繋がっているか。途中で論点がすり替わっていないか（例: NG「道具が大事」→「レシピだけ渡される」= 論点が2つ混在）
> 2. **初心者読解テスト**: 書いた比喩を「AIもプログラミングも知らない人」の目線で読み直す。「で、つまり何？」と思わないか。比喩の結論が**伝えたいメッセージ（環境が悪い等）**にストレートに着地しているか
> 3. **軸統一テスト**: 台本内で同じ比喩を複数回使う場合（例: 料理の比喩）、全箇所で**同じ世界観・同じキャラクター・同じ対比構造**を使っているか。途中で比喩の軸がブレていないか
3. **数字は具体的に**: 「たくさん」ではなく「80%」「3万円」など具体数字にする
4. **一番理解力の低い人でもわかるレベル**で書く

### 構造

1. **タイムスタンプを付ける**: 各セクションの先頭に `[MM:SS]` を表記する
2. **セクション見出しを付ける**: `## セクションN: [名前]` の形式で見出しを入れる
3. **MPCSループのフェーズを明示する**: `【未来】`、`【問題】`、`【原因】`、`【解決】` のラベルを台本内に挿入する
4. **テーマ回帰の台詞を入れる**: 構成表で指定されたポイントに「覚えてますか？今日のテーマは○○でしたよね」を挿入する
5. **スライド切り替え指示を入れる**: `📊 [スライド: ○○]` の形式でスライドの切り替えタイミングを表記する
6. **スライドは最大21枚（NotebookLM制限）**: スライドは喋りの「補助」。話者がメインで伝えるため、視覚的な支援が必要な箇所だけにマーカーを置く。以下を基準に絞り込む:
   - ✅ 残す: **数字の提示**（損失額、時間等）、**対比の図解**（VS構図、ビフォーアフター）、**チェックリスト**（持ち帰り品の確認）、**具体例の箇条書き**、**テーマ提示**
   - ❌ 削る: **感情訴求**（絶望・共感 → 話者の声のトーンで伝える）、**シンプルな質問**（エンロール等 → 話者が直接聞く）、**短いフレーズ1つだけのスライド**
   - 🔀 統合: 同じ文脈の連続スライド（自己紹介+実績、99%+Google製 等）は1枚にまとめる

### Markdownフォーマット

1. **本文に太字（`**〜**`）を使わない**: 台本の本文（話す言葉）はプレーンテキストで書く。アスタリスクで強調しない。話者が声のトーンで強調するので、視覚的な太字装飾は不要
2. **スライド行は太字にする**: `📊 [スライド: ○○]` の行は必ず `**📊 [スライド: ○○]**` のように太字で囲む。台本内でスライド挿入ポイントが一目でわかるようにする
3. **太字を使ってよい箇所**: メタデータ（テーマ/時間/対象）、セクション見出し（##, ###）、`→ **テーマ回帰**:` の行のみ

### 定義の扱い

重要キーワードが登場する箇所では、以下の順序で書く:

```
「じゃあ、○○って何だと思いますか？ちょっと考えてみてください。

○○って、一言で言うと△△なんですよ。

例えば〜（具体例で噛み砕く）〜

だから○○が分かれば〜（展開）〜」
```

### 定義の復唱確認 ★NEW

定義を一度共有した重要キーワードは、ループの切り替え時やCTA前に**復唱を求める**台詞を入れる:

```
「覚えてますか？○○の定義は何でしたか？

…そう、△△ですよね。これ、めちゃくちゃ大事なので忘れないでください。」
```

### 導入の流れ（7ステップ）

台本の冒頭は**挨拶や自己紹介から入らない**。
以下の順序で、聴衆を自然にテーマに引き込む:

**Step 1: 引きの言葉（フック）**
聴衆の心を掴む衝撃的な事実やストーリーで開始する。

**Step 2: エンロールクエスチョン**
「○○したい人いますか？」「○○って感じてる人は？」で聴衆の共感を取る。

> **❗ 感情継続の原則**: Step 1のフックで作った感情（痛み、驚き、共感）を**そのままエンロールの燃料に使う**。
> - 「ちなみに」「ところで」等のトピック転換語で感情を断ち切らない。
> - フックの余韻を残したまま「でもね、こう思ったことないですか？」で自然に橋渡しする。
> - 原則: **ストーリーの感情 → 「あなたも同じでしょ？」 → 共感** の一本線で繋ぐ。

**Step 3: テーマのキーワード定義共有**
テーマに関連する核心キーワードの定義を、この段階で先に共有する。
「ところで、○○って何だと思いますか？…○○っていうのは、一言で言うと△△なんです」
→ 定義を共有してから話を展開する（定義先行の原則）

**Step 4: 未来の提示**
その定義を使って「もしこうなったらどうですか？」と未来像をコンパクトに見せる。
導入なので深く描かず、ダイジェストで見せるだけ。ループ1で深掘りする。

**Step 5: 興味確認**
「…こんな未来、興味ないですか？」で聴衆の反応を確認する。

**Step 6: 3つの持ち帰り品宣言**
未来に興味を持った状態で「今日、まさにその未来に近づくための3つが手に入ります」と宣言する。
→ 唐突に3つを出さず、**未来提示→興味確認→だからこの3つ** の流れを必ず守る。

**Step 7: テーマ提示**
テーマ「○○からの脱却」を**1文だけ**で宣言する。
→ **テーマは必ず1つだけ**。補足説明は加えない。テーマ宣言が2文以上になると「テーマが2つある」ように聞こえるので厳禁。

```
「（Step 1: フック — 衝撃的な数字、ストーリーの冒頭など）

…でもね。皆さん、こう思ったことないですか？（Step 2: エンロール — フックの感情をそのまま使う）
「○○って思ったことがある人、いますか？」
ありがとうございます。

ところで、○○って何だと思いますか？（Step 3: 定義共有）
○○っていうのは、一言で言うと△△なんです。

で、この○○を手に入れたら、どうなるか。（Step 4: 未来提示）
（未来像をコンパクトに提示）

…興味ないですか？（Step 5: 興味確認）

今日、まさにその未来に近づくための3つが手に入ります。（Step 6: 3つの持ち帰り品）
（1）○○がわかる。
（2）○○がわかる。
（3）○○ができるようになる。

今日のテーマはこれです。「○○からの脱却」。（Step 7: テーマ提示 — 1文で）」
```

**引きの言葉のパターン:**
- 衝撃的な数字: 「150時間。これ、僕がドブに捨てた時間です」
- 質問形式: 「皆さん、AIって使いこなせてますか？…正直に言って、使いこなせてる人って何%くらいだと思います？」
- ストーリー冒頭: 「画面の前で、僕は呆然としてました。150時間かけて作ったものが、一瞬で消えたんです」
- 意外な事実: 「99%の人がまだ知らないツールがあります。しかもGoogleが作った、無料のツールです」

### 自己紹介の位置

自己紹介は**導入の直後、MPCSループ1の前**に置く。

目的: 聴衆が「うんうん、興味ある！」となった後に、**「なぜこの人の話を聞くべきなのか？」** の理由付けを行う。

```
「で、「誰がそんなこと言ってるの？」って話ですよね。

自己紹介させてください。○○です。（簡潔な自己紹介）

…って言うと「すごい」って思われるんですけど。
X年前は、さっき話した「○○」です。（聴衆と同じ側にいた過去）

でも、○○したら、人生がガラッと変わりました。（変化のビフォーアフター）

だから今日は、皆さんにもその話をしたいんです。」
```

→ 自己紹介は**2分以内**に収める。長くなりすぎない。
→ 「昔は聴衆と同じ側にいた」→「でも変化した」のストーリーで信頼を構築する。

### MPCSループの濃度（最重要）

**MPCSの各パートは「薄く説明する」のではなく「聴衆の心を動かす」レベルで書く。**

各パートの濃度基準:

1. **未来(M)**: 手段（サイトが作れる、ツールが作れる）ではなく、**人生の成果**で語る。聴衆が「その未来欲しい！！」と心から思うべネフィットを描く。例:
   - 時間が浮く→旅行、家族との時間、婚活、趣味
   - 副収入が増える→副業で月♅5万〜10万
   - キャリアアップ→「AI人材」として会社で重宝される、転職の武器
   - 自信がつく→「自分にもできる」の全能感
   五感に訴える。「朝起きて」「上司に見せて」「通帳を見て」のように行動レベルで描く

2. **問題(P)**: 聴衆が「それは本当になりたくない！」と痛みを感じるほどの絶望を描く。具体的なシーンの再現に加えて、**「放置した未来」**も描く:
   - 「このまま何もしなかったら、5年後どうなる？」
   - 「まわりは使いこなしているのに、自分だけ取り残される」
   - 「どっちに転んでも絶望」（使おうとしたら挫折、使わなかったら取り残される）
   抽象的な問題ではなく、「あの時の、あの画面。エラーメッセージが赤く光ってて。何回やっても同じ」のレベルで書く

3. **問題(P)の構造的原因指摘 ★NEW**: 「あなたは悪くない」の後を、抽象的な「環境が悪い」で止めない。具体的にどの仕組み/業界構造/教える側が悪いのかを名指しに近いレベルで指摘する:
   - 「YouTubeでもネットでも、AI使おうって教えてるけどさ、教えてる人自身がエンジニアだから、そもそもプログラミング知識ゼロの初心者の気持ちなんて分かってないんですよ」
   - 「AI系のツールを作ってる会社だって、ターゲットはエンジニアですよ。初心者のことなんて考えてないんです」
   - 「プログラミングスクールだってそう。20万〜100万とって教えるけど、そもそもスキル習得に何ヶ月もかかる前提のビジネスモデルですよね」
   共通の敵を作ることで、聴衆との一体感を生む

4. **原因(C)**: 「実は原因は○○です」だけで終わらせない。なぜそれが原因なのかを、比喩と第三者事例を使って多角的に説明する。聴衆が「そういうことだったのか！」と目が見開く気づきのシーンを作る。最低200字以上は使う。**第三者事例は各ループに必ず1つ以上**:
   - 研究データ: 「〇〇大学の調査では〜」「経産省のデータによると〜」
   - 統計・数字: 「AIスキル人材の平均年収は+200万円高い」「AI活用企業の生産性は1.5倍」
   - 他者の成功事例: 「受講生のAさんは〜」「この企業は〜」
   - 業界の意外な事実: 「西陣織の廃材が着火剤で700円で売れている」

5. **解決(S)**: 解決策を提示した後、その解決策がもたらす変化をビフォーアフターで見せる。数字で示す。「乗ってみさえすれば」「やってみさえすれば」の表現で、**環境を手に入れるだけでなく「行動すること」が変化のトリガー**であることを示す

**1つのMPCSループの目安文字数:**
- 10分台本: 1ループ 800〜1,200字
- 30分台本: 1ループ 2,000〜3,000字
- 60分台本: 1ループ 3,000〜4,000字

### ワーク（参加行動）★NEW

台本中に最低1回は「聴衆が手を動かす・考える・書き込むワーク」を挿入する。

```
「ちょっとここでワークをやりましょう。
[具体的な指示: 思い浮かべて/チャットに書いて/隣の人と話して]
…どうですか？[感想を引き出す質問]
ありがとうございます。」
```

挿入タイミング:
- ループ2の後（気づきを体感させる）
- CTA直前（当事者意識を高めた状態で行動を促す）

### 損失額の具体化と「ギャップ＝価値」フレーム ★NEW

CTA直前のセクションで、聴衆がすでに失っている「時間」「お金」の累計損失額を具体的に算出して見せる:

```
「ちょっと計算してみてください。
毎月X時間、ムダにしてるとしますよね。年間でY時間。時給Z円で換算したら、年間W万円の損失ですよ。
3年続けたら…○万円。
でもこの教材は○○円。ランチ1回分。
この差、分かりますか？
…変化の幅が大きいほど、それに価値があるんです。」
```

### CTA（教材案内 + メルマガ導線）

- 教材の機能・内容・カリキュラムは**絶対に説明しない**
- 「今日の気づきをさらに深めたい方は」「もっと詳しく知りたい方は」の流れで自然に誘導する
- クロージングのメッセージは「環境があれば」ではなく「**乗ってみさえすれば**」「**やってみさえすれば**」と行動を促す表現を使う。正しい自転車を手に入れただけでは話にならない。乗ることが大事
- **限定性・緊急性の演出を必ず1つ以上入れる**: 「今日だけ」「限定○名」「メルマガ登録者限定価格」「特典付き」
- **メルマガ導線 ★NEW**: CTAの最後に、メルマガ登録への導線を入れる:

```
「そしてもう1つ、大事なお知らせがあります。

僕のメルマガに登録していただくと、確実に1000円台でゲットできるご案内が届きます。

今日ここで見ていただくのももちろんいいんですけど、メルマガに入ってもらえれば、最安値で確実にゲットできるご案内が届くので。

QRコードはこちらです。」
```
