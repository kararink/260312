# 自治体DX推進とLGWAN・DAP活用に関するブリーフィング資料

## エグゼクティブ・サマリー

本資料は、地方自治体におけるDXの基盤となる**「LGWAN（総合行政ネットワーク）」**の現状と、その制約を解消し活用を加速させる**「DAP（デジタルアダプションプラットフォーム）」**の重要性について構造化したものである。

地方自治体は、高度なセキュリティ（三層分離モデル）を維持しつつ、行政事務の効率化と利便性向上を迫られている。特に、改修困難な国の基幹システム（JCIP/eMLIT等）における操作迷いや申請不備は、職員・住民双方の大きな負担となっている。

この課題に対し、既存システムを改修せずブラウザ上でリアルタイムなガイドを提供する**DAP（特に「テックタッチ」）**の導入が、兵庫県を含む先行自治体で顕著な成果（操作時間短縮、不備削減等）を上げている。今後は生成AIとの連携により、事務支援のさらなる高度化が期待される。

---

## 1. LGWAN（総合行政ネットワーク）の構造と特性

### 1.1 概要と基本方針
LGWAN（Local Government Wide Area Network）は、2001年度より運用されている地方公共団体相互間の行政専用クローズドネットワークである。以下の基本方針に基づき、安全かつ標準的な通信基盤を提供している。

- **高度な安全性**: インターネットから物理的・論理的に分離。
- **全自治体の包含**: 規模や地域を問わず、全地方自治体を接続。
- **標準技術と低コスト**: 汎用技術の採用により負担可能なコストを実現。
- **政府接続**: 政府共通ネットワークと相互接続し、国・地方間の連携を担保。

### 1.2 技術的構成要素
| 構成要素 | 概要 |
| :--- | :--- |
| **LGWAN接続ルーター** | 庁内LANとLGWANを接続するゲートウェイ。 |
| **都道府県ノード** | 各都道府県に設置された100Mbps～1Gbpsの中継設備。 |
| **POI (Point of Interface)** | 東西日本に分散設置された監視・基本プロトコル提供設備。 |
| **NOC (Network Operation Center)** | J-LIS（地方公共団体情報システム機構）による全体管理設備。 |

### 1.3 三層分離モデルによるセキュリティ
2015年の事案以降、自治体ネットワークは強固な「三層分離」が維持されている。

1.  **マイナンバー利用事務系**: 社会保障・税等の最重要領域。
2.  **LGWAN接続系**: 財務会計・人事給与等の主要事務領域。
3.  **インターネット接続系**: 情報収集・クラウド利用。
    - *補足*: 現在は利便性向上のため、特定のクラウドへ直接接続する「α'モデル」等の改善が進んでいる。

## 2. 自治体DXを支援するDAP（デジタルアダプションプラットフォーム）

### 2.1 DAPの定義と必要性
DAPは、新しい技術やシステムをユーザーが直感的に操作できるよう、画面上で「伴走型支援」を行うツールである。

- **システムの複雑性**: JCIP（建設業許可）やeMLIT（宅建業免許）等の膨大な入力項目への対応。
- **運用の継ぎ目**: 国のシステムと自治体独自の電子納付システム間の操作迷い。
- **人的コストの削減**: 申請不備（リワーク）や電話問い合わせ対応の抜本的解消。

### 2.2 テックタッチの主要機能
既存システムのソースコードを一切書き換えることなく、以下の機能をブラウザ上で実装可能。

- **ステップナビゲーション**: 操作フローをリアルタイムで画面上に表示。
- **入力バリデーション**: 全角・半角、桁数等のエラーミスを即座に通知。
- **ツールチップ**: 専門用語に対するマウスオーバー注釈。
- **アナリティクス**: ユーザーの離脱ポイントをデータ化し、改善を高速化。

## 3. 兵庫県におけるDX推進とDAP活用戦略

### 3.1 スマート兵庫戦略
行政手続のオンライン利用率**85.2%**の達成を目標に、全域でのデジタル実装を加速中。

### 3.2 兵庫県電子申請システムへの導入実績
「不妊治療助成申請」において先行導入。
- **課題**: 年間約3,300件の申請中、過半数で不備が発生。
- **解決**: テックタッチによるリアルタイムガイド。
- **効果**: 審査負担の軽減と、受給者への迅速な支給を実現。

### 3.3 国の基幹システムへの展開（検討中）
自治体側で改修不能なシステムに対し、**ブラウザ拡張機能**を用いたDAP実装を検証中。

| 対象システム | 狙い |
| :--- | :--- |
| **JCIP（建設業許可）** | 財務諸表等の専門的入力における不備削減。 |
| **eMLIT（宅建業免許）** | 県独自の電子納付システムとの連携における操作迷いの解消。 |

## 4. 自治体・官公庁における導入成果（事例）

| 導入主体 | 成果の概要 |
| :--- | :--- |
| **宇都宮市** | 電子申請システムの操作時間を**約63%短縮**。 |
| **神戸市** | 問い合わせ件数を**約40%削減**。FAQ閲覧数2.3倍。 |
| **デジタル庁** | 入札手続きのミス防止。調達ポータルの業務負荷軽減。 |
| **島根県** | 次期財務会計システム（220部署）での用語不一致をカバー。 |

## 5. 今後の展望：生成AIとの融合

DAPは「受動的なガイド」から、AIによる**「能動的な伴走（AIエージェント）」**へと進化している。

- **Techtouch AI Hub**: 法令や規定準拠をAIが自動判断。
- **自動バリデーション**: 添付書類（納税証明書等）と入力値の突合による不備防止。
- **ホスピタリティの追求**: デジタルデバイド対策と効率化を両立し、住民サービスの質を根本から引き上げる。
