# STEPN 自動SNS投稿システム 構想・設計図 (Draft)

このドキュメントは、STEPNの記録を基にした自動SNS投稿システムの要件定義および設計を詰めていくためのベースファイルです。まずは開発を進めず、ここで「何を・どのように・なぜ」作るのかを徹底的に議論し、構造化します。

---

## 1. プロジェクト概要 (Project Overview)
*   **目的・ゴール**: STEPNの活動記録（ジョギング等）をトリガーとし、自動的または半自動的にSNSへコンテンツを発信する仕組みを構築する。
*   **対象プラットフォーム**: X（Twitter）、Facebook、Instagram
*   **期待する効果**:
    *   毎日のルーティン（Sensory Grounding）の可視化と継続の促進
    *   「1年前の自分と比べてできるようになったこと」の絶対評価の記録
    *   SNS運用コストの極小化（無為自然・省力化の実現）

## 2. 杢之助の文脈との統合 (Context & Tone)
*   **発信のスタンス**:
    *   「WHAT重視（Fact over Theory）」: 一次データ（STEPNの距離、時間、獲得GST等の客観的数値）をベースにする。
    *   「科学的リアリズム」: 感情論ではなく、ジョギングによるメンタル・フィジカルへの影響など、客観的・メタ的な視点を取り入れる。
    *   「Architect（建築家）」のトーン: 冷静沈着で、過剰な装飾や感傷を排し、事実とそれに基づく考察を淡々と綴る。
*   **ターゲット**: 過去・未来の自分自身、あるいは同じようにルーティンを大切にする人々。承認欲求（SNS的称賛）を目的としない。

## 3. システム要件 (System Requirements)
### 3.1. インプット & トリガー (Input & Trigger: Pixel 9 / Android環境)
Androidの強みである「カスタマイズ性」を活かし、以下の手法を採用する。
*   **STEPNアプリの「シェア（共有）」機能の拡張**:
    *   MacroDroidやTasker等のAndroid自動化アプリを使用し、共有メニューに「SNS自動投稿システム」宛ての専用ボタンを作成する。
    *   共有された画像を、システム側（Google Drive、またはWebhook経由で直接）へ転送する。

### 3.2. 処理・生成 (Processing & Generation)
*   **データ抽出 (OCR / Vision AI)**:
    *   共有された画像（スクリーンショット）から、距離（km）、時間、獲得GST、消費エネルギー等の数値を抽出する。
    *   ※Kindle OCRでの知見、または最新のVision AI（Gemini等）を用いることで高精度の抽出が可能。
*   **テキスト生成 (定型文 ＋ AIアレンジ)**:
    *   抽出した数値をシステム側で変数として扱い、事前に用意した定型文（例：「{距離}kmステップンしたよ」）に埋め込む。
    *   プラットフォーム（X, Facebook, Instagram）ごとの文字数やトーンに合わせて、システムがテキストを最適化する。

### 3.3. アウトプット (Output)
*   **X (Twitter)**: 短文（「〇〇km走りました」等の定型文＋抽出ファクト＋ハッシュタグ）
*   **Facebook**: 少し長文、インサイトや記録としての文脈を付与したフォーマット
*   **Instagram**: STEPNの画像をメインとし、キャプションにAI生成テキストを配置

## 4. ワークフロー設計 (Workflow Design)
早朝のチャロちゃんとの散歩時（PC電源オフ・Wi-Fi持ち出し外）を前提とした、**「完全無人・全自動」**のフローとなります。

1.  **[走る＆共有]**: STEPN終了後、「シェア」ボタンからAndroidの専用自動化アプリ（MacroDroid等）を選択、または特定フォルダへ画像を保存する。
2.  **[クラウド転送]**: スマホのモバイル回線経由で、画像データがクラウド（Google Drive等）または連携用Webhookへ送信される。
3.  **[抽出＆生成＆自動投稿]**: 
    *   クラウド上のシステム（MakeやGAS）が発火。
    *   画像から数値を読み取り（OCR等）、定型文に数値を当てはめた投稿文をAIが作成。
    *   **そのままX、Facebook、Instagramの3媒体へ一斉に自動連携・投稿**される。

## 5. アーキテクチャ (Architecture: GAS / フルスクラッチ)
「コストゼロ」と「自作の知見（資産化）」を重視し、**Google Apps Script (GAS)** をハブとするフルスクラッチ構成を採用する。

### 5.1. 技術要素
*   **スマホ側 (Android)**: MacroDroid もしくは Tasker
    *   STEPNからの画像を受け取り、GASのWebアプリURL（doPostエンドポイント）へHTTP POSTする。
*   **バックエンド (GAS)**:
    *   **フロント（受信）**: `doPost`関数によるWebhookエンドポイント。
    *   **データ抽出**: 画像解析API（Google Cloud Vision API、またはGemini APIのVision機能）を利用し、画像から数値を抽出。
    *   **テキスト処理**: 抽出データを定型文（テンプレート）に代入し、各媒体向けのテキストを生成。
*   **SNS連携 (API連携)**:
    *   **X (Twitter)**: X API v2を利用。
    *   **Facebook / Instagram**: Meta Graph APIを利用。
    *   ※いずれもGASからAPIエンドポイントへ適切な認証情報（アクセストークン等）を付与してリクエストを送信するプログラムを記述する。

### 5.2. 懸念点・リスク管理
*   **API認証の壁**: 特にMeta（Facebook/Instagram）のAPIはアクセストークンの有効期限管理（自動更新処理）等、独自の実装の手間が大きい。
*   **画像抽出の精度**: OCRAPIの精度により読み取りミスが発生する可能性があるため、GAS上で正規表現等を用いた強力なバリデーション（データチェック＆補正）処理を入れる必要がある。

## 6. 開発フェーズ (Development Phases)
今後の開発手順（タスクブレイクダウン）は以下の通り。開発はこの順序に沿って、ステップ・バイ・ステップで進めていく。

1.  **Phase 1: インプットの確立 (Android → GAS)**
    *   Androidの自動化アプリで「画像を受け取り、特定URLに投げる」処理を作る。
    *   GASで画像を受信し、Google Driveへ保存できるか（疎通確認）をテストする。
2.  **Phase 2: データ抽出の確立 (GAS)**
    *   受信した画像をAPIに渡し、期待する数値（距離、時間、GST等）が正しく取り出せるロジックを組む。
3.  **Phase 3: テキスト生成の確立 (GAS)**
    *   抽出した数値を定型文に流し込み、テキストを生成するロジックを組む。
4.  **Phase 4: アウトプットの確立 (GAS → 各SNS)**
    *   最も難易度の高い部分。各SNSのアカウントとAPI連携し、GASから投稿できるスクリプトを個別に構築・テストする。
5.  **Phase 5: 統合・総合テスト**
    *   Androidからの画像一枚で、3媒体すべてに投稿されるかを実証する。
